Documentation for the "whereFields" JSON key.

whereFields allow binding of parameters to the fields mentioned in the whereClause key.

whereFields are used within multiple SQL interactors and are in one of two formats.

The first format is a JSON array of String field names which can also crosstable refenerce other fields.

The second format is a JSON Array of JSON Objects which contain a name and an aliaskey. This is used where the field being parameterised is used in two different ways. See the examples at the bottom of the page for usage information.

The where field can also be used with the in: designator to provide a list of multiple values.

Allowable usage within SQL interactors for a whereField JSON Object/Array

action nameusage
insertersoptional (only valid in conjunction with whereClause)
updatersoptional (only valid in conjunction with whereClause)
deletersoptional (only valid in conjunction with whereClause)
findersoptional (only valid in conjunction with whereClause)
questionsoptional (only valid in conjunction with whereClause)
countersoptional (only valid in conjunction with whereClause)

The whereFields are a list of the name and type for each of the whereClause parameters.

If there is a whereFields key on the JSON object without any whereClause then this will generate a FATAL validation message.

Example 1: Simple whereFields example

The where fields reference the table to which they are attached. The lookup of the field definitions are automatically synchronised, such that the type is inferred from the table.

{
	...

	"whereClause": "where something = ? and something_else = ?",
	"whereFields": [
		"something",
		"something_else"
	]
}

Example 2: Simple whereFields example, with cross-referenced tables

The where fields can also look up the details from other tables within the h2zero file. This can be accomplished by referencing the fully qualified table name and field name, as shown below.

The some_other_table.something_else designator will look up the something_else field definition on the some_other_table table definition.

{
	...

	"whereClause": "where id_something = (select id_something from some_other_table where something_else = ?)",
	"whereFields": [
		"id_something",
		"some_other_table.something_else"
	]
}

Example 3: Aliased whereFields example

The where fields reference the table to which they are attached, however, the referenced field is used in two different contexts. Consequently two parameters are required to be passed, separately named.

{
	...

	"whereClause": "where something > ? and something < ?",
	"whereFields": [
		{ "name": "something", "alias": "from" },
		{ "name": "something", "alias": "to" }
	]
}

Example 4: Using the in: designator

The in: designator tells h2zero that an ArrayList of values will be passed in to the method. The field reference will be looked up on the table to which this is attached.

{
	...

	"whereClause": " where id_something in (...)",
	"whereFields": [
		"in:id_something"
	]
}
