Question.

Questions are simple SQL statements that return one, and only one boolean (true/false) value.

This is a light-weight way to get a boolean answer to a question without using select beans or counters

The key questions is a JSON Array on the table JSON object, for example:

"tables": [
	{
		"fields": [
			...
		],
		"questions": [
			...
		]
	}
]

Allowable keys for a Question JSON Object

key namekey value typeusagenotes
nameStringmandatoryThis is the name for the question. If not present on the JSON object an H2ZeroParseException will be thrown.
selectClauseStringmandatoryThis is the SQL query that is run.

The select clause statement MUST return one and only one boolean result.

This will generate a FATAL validation message if not present on the JSON object.
selectFieldsJSON ArrayinvalidSelect fields are invalid for the Question JSON object.

The select clause is automatically generated with a single select field (which is of type boolean).

This will generate a FATAL validation message if present on the JSON object.
whereClauseStringoptionalUsed if the SQL query is parameterised.
whereFieldsJSON ArrayoptionalThe name and type for each of the whereClause parameters. If there is a whereFields key on the JSON object without any whereClause then this will generate a FATAL validation message.
uniquebooleanignoredWhether the SQL query will return a single object or an ArrayList of objects.

This always returns one, and only one result in the result set of type boolean and is therefore un-necessary.

This will generate a WARN validation message if present on the JSON object.

Example 1: Simple Question

"questions": [
	{
	"name": "doWeHaveMoreThanTwentyUsers",
	"selectClause": "select count(*) > 20 from user"
	}
]

Whilst rather contrived, this will return a true/false answer as to whether we have more than 20 users.

You may also use a whereClause and whereFields if you wish to pass in additional parameters.

Example 2: Parameterised Question

"questions": [
	{
		"name": "doesUserNameExist",
		"selectClause": "select count(*) > 0 from user",
		"whereClause": "where nm_user = ?",
		"whereFields": [
			"nm_user"
		]
	}
]

In the above example, the question being asked is whether a particular username exists in the user table. If it does, the return value is true else false.
